## Overview

The `@raypx/rag` package provides document parsing, chunking, and vectorization capabilities for building RAG (Retrieval-Augmented Generation) applications. It supports multiple embedding providers and can process various document formats.

## Features

- **Document Parsing**: Extract text from PDF, DOCX, and text files
- **Text Chunking**: Split documents into smaller chunks with overlap
- **Embedding Generation**: Generate embeddings using multiple providers (OpenAI, Hugging Face, Cohere, DeepSeek, Aliyun)
- **Vector Storage**: Store embeddings in PostgreSQL with pgvector

## Installation

The `@raypx/rag` package is already included in the workspace. No additional installation is required.

## Supported Providers

### 1. OpenAI
- Models: `text-embedding-3-small`, `text-embedding-3-large`, `text-embedding-ada-002`
- Dimensions: 1024 (small), 3072 (large), 1536 (ada-002)
- Cost: Pay per token

### 2. Hugging Face
- Models: Any sentence-transformers model
- Popular: `sentence-transformers/all-MiniLM-L6-v2` (384 dims), `BAAI/bge-large-en-v1.5` (1024 dims)
- Cost: Free (with API key) or self-hosted

### 3. Cohere
- Models: `embed-english-v3.0`, `embed-multilingual-v3.0`
- Dimensions: 1024
- Cost: Pay per request

### 4. DeepSeek
- Models: `text-embedding-3-small` (fallback, DeepSeek may not support embeddings)
- Dimensions: Configurable
- Cost: Pay per token (typically cheaper than OpenAI)
- **Note**: DeepSeek may not support embeddings API, consider using other providers

### 5. Alibaba Cloud / DashScope (阿里云)
- Models: `text-embedding-v1`, `text-embedding-v2`, `text-embedding-v3` (通义千问)
- Dimensions: Configurable (typically 1536 for v3)
- Cost: Pay per token (competitive pricing, good for Chinese users)
- Note: Uses OpenAI-compatible API format through DashScope compatible mode
- API Endpoint: `https://dashscope.aliyuncs.com/compatible-mode/v1`

## Configuration

### Environment Variables

#### Provider Selection
```bash
EMBEDDING_PROVIDER=openai  # Options: openai, huggingface, cohere, deepseek, aliyun
EMBEDDING_API_KEY=your-api-key  # Unified API key for all providers
EMBEDDING_API_URL=https://custom-endpoint.com  # Optional, for self-hosted models
```

#### Model Configuration
```bash
EMBEDDING_MODEL=text-embedding-3-small  # Provider-specific model name
EMBEDDING_DIMENSIONS=1024  # Optional, will use provider default if not set
```

#### Chunking Configuration
```bash
CHUNK_SIZE=1000  # Chunk size in characters (default: 1000)
CHUNK_OVERLAP=200  # Chunk overlap in characters (default: 200)
```

### Provider-Specific Configuration

#### OpenAI
```bash
EMBEDDING_PROVIDER=openai
EMBEDDING_API_KEY=sk-...
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_DIMENSIONS=1024
```

#### Hugging Face
```bash
EMBEDDING_PROVIDER=huggingface
EMBEDDING_API_KEY=your-hf-token  # Optional, for rate limits
EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2
EMBEDDING_API_URL=https://your-self-hosted-endpoint.com  # Optional
```

#### Cohere
```bash
EMBEDDING_PROVIDER=cohere
EMBEDDING_API_KEY=your-cohere-key
```

#### DeepSeek
```bash
EMBEDDING_PROVIDER=deepseek
EMBEDDING_API_KEY=your-deepseek-key
EMBEDDING_MODEL=text-embedding-3-small  # Fallback model
EMBEDDING_DIMENSIONS=1024
```

**⚠️ Warning**: DeepSeek primarily provides chat models and may not support embeddings API. If you encounter `MODEL_NOT_FOUND` (404) errors, consider using other providers.

#### Alibaba Cloud / DashScope
```bash
EMBEDDING_PROVIDER=aliyun
EMBEDDING_API_KEY=your-dashscope-key
EMBEDDING_MODEL=text-embedding-v3  # Default: text-embedding-v3
EMBEDDING_DIMENSIONS=1536  # Optional, v3 default is 1536
```

## Usage

### Vectorize a Document

```typescript
import { vectorizeDocument } from "@raypx/rag";

const result = await vectorizeDocument(documentId, userId, {
  chunkSize: 1000,
  chunkOverlap: 200,
  embeddingProvider: "openai", // or "huggingface", "cohere", "deepseek", "aliyun"
  embeddingModel: "text-embedding-3-small",
  embeddingDimensions: 1024,
});

console.log(`Created ${result.chunksCreated} chunks and ${result.embeddingsCreated} embeddings`);
```

### Generate Embeddings

```typescript
import { generateEmbeddings } from "@raypx/rag";

const embeddings = await generateEmbeddings(texts, {
  provider: "openai", // or "huggingface", "cohere", "deepseek", "aliyun"
  model: "text-embedding-3-small",
  dimensions: 1024,
});
```

## Supported File Types

- PDF (`.pdf`)
- Word Documents (`.docx`, `.doc`)
- Text files (`.txt`, `.md`, `.json`, `.xml`, `.csv`, `.html`, `.htm`)

## Architecture

1. **Parsers**: Extract text from various file formats
2. **Chunking**: Split text into manageable chunks with overlap
3. **Embedding**: Generate vector embeddings using LangChain
4. **Storage**: Store chunks and embeddings in PostgreSQL with pgvector

## Automatic Retry

Failed vectorizations are automatically retried via a cron job endpoint:

- **Endpoint**: `/api/cron/retry-vectorization`
- **Schedule**: Configure via external cron service

The retry system:
- Processes documents with `failed` or `uploaded` status
- Respects retry limits (default: 3 attempts)
- Maintains minimum retry delay (default: 1 minute)
- Logs retry attempts and errors in document metadata

## Error Handling

Vectorization errors are logged with detailed information:
- Document ID and user ID
- Error message and stack trace
- Provider and model information
- Document status is updated to `failed` on error

## Best Practices

1. **Choose the right provider**: 
   - Use OpenAI for high-quality embeddings
   - Use Hugging Face for free/open-source options
   - Use Alibaba Cloud for Chinese users (low latency, competitive pricing)
   - Use Cohere for multilingual support

2. **Optimize chunk size**:
   - Smaller chunks (500-1000 chars) for precise retrieval
   - Larger chunks (1000-2000 chars) for context preservation
   - Use overlap (10-20%) to maintain context across chunks

3. **Monitor retries**:
   - Check document metadata for retry counts
   - Review failed documents regularly
   - Adjust retry limits based on your needs

4. **Cost optimization**:
   - Use appropriate dimensions for your use case
   - Consider self-hosted models for high-volume usage
   - Monitor API usage and costs
